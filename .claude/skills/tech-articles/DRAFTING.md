# Drafting reference

## The 4 signature angles

Pick ONE lead angle per article (may blend a second lightly). This is the differentiator.

| Angle | Core move | Opening shape |
|-------|-----------|---------------|
| **story-driven** | Problem → struggle → insight arc, first person | "We had X. It broke when Y. Here's what I missed." |
| **teach-by-pattern** | Extract one reusable principle reader applies elsewhere | "There's a pattern behind this. Steal it." |
| **contrarian** | Challenge a common practice with evidence | "Everyone says do X. In this project X cost us Y." |
| **real-code-tradeoff** | Show the actual decision + what was given up | "We chose A over B. Here's the tradeoff nobody mentions." |

Every draft must carry its angle through the ending, not just the hook.

## Structure (both platforms share this spine)

1. **Hook** (first 1–2 lines): concrete tension or claim. No warmup. Must survive being the only thing shown before "…more".
2. **Context**: the real situation in 1–3 lines. Name the actual thing (feature, migration, bug).
3. **Meat**: the decision/pattern/tradeoff. Include ONE real code detail or concrete number — traceable to the repo.
4. **Turn**: the insight, the "so what" for the reader.
5. **Takeaway**: one line the reader can reuse. Optional soft question to drive comments.

## LinkedIn — compact essay

- 120–250 words. Short paragraphs (1–3 lines each). Whitespace between.
- Plain language, senior-dev tone. Mixed audience: devs, clients, IT folks — explain jargon in half a sentence.
- Max 1 code snippet, ≤8 lines, only if it earns its place. Prefer describing the decision.
- 3–5 relevant hashtags at end. No emoji spam (0–2 max, purposeful).
- End with takeaway + optional question.

## Twitter/X — thread

- Tweet 1 = hook, standalone, ≤270 chars, must make scroll stop. No "a thread 🧵" filler unless it adds nothing to cut it.
- 4–8 tweets. One idea per tweet. Each tweet readable alone.
- Code as a short snippet in its own tweet only if crucial.
- Last tweet = takeaway + soft CTA (follow / reply).
- No hashtag stuffing (0–2, in last tweet).

## Voice consistency

- Load [voice-profile.md](voice-profile.md). Apply niche framing, honor hot-take stances, avoid banned words.
- Keep same person/tense across a post. Default first person for story/tradeoff, second person ("you") for pattern/contrarian teaching.

## Quality gate before delivering

- [ ] Hook works with zero context
- [ ] At least one concrete, repo-traceable detail (not generic)
- [ ] One clear takeaway
- [ ] Angle carried hook → ending
- [ ] Banned words absent, jargon explained
- [ ] LinkedIn ≤250 words; thread tweets ≤270 chars each
